<pitfall_5_enforcement>
**CRITICAL: this plan exists BECAUSE of Pitfall #5. Read this block before any task.**

The whole point of the test-connection button is to prove that the production routing path works end-to-end for a specific org. That requires routing through the EXACT SAME code the scanner uses:

```
POST /admin/organizations/:id/branding-test
   → server.brandingOrchestrator.matchAndScore({ orgId, siteUrl, scanId, issues, guideline })
      → orgRepository.getBrandingMode(orgId)    ← per-request read, mode dispatch
      → if mode==='remote' → remoteAdapter.matchForSite  ← HTTP call to @luqen/branding
        else              → embeddedAdapter.matchForSite ← in-process BrandingMatcher
      → calculateBrandScore(brandedIssues, guideline)    ← pure score
   → returns { kind: 'matched'|'degraded'|'no-guideline', mode, ... }
```

**PROHIBITED shortcuts (each would silently mask a dual-mode routing bug):**

1. DO NOT call `brandingService.listGuidelines()` directly. That hits `/api/v1/guidelines` on the branding service, bypassing the orchestrator entirely, skipping the per-request mode read, and producing a "test" result that has no relationship to whether a scan on this org would actually succeed. This is the specific anti-pattern Pitfall #5 names.
2. DO NOT call `fetch(brandingUrl + '/api/v1/health')`. Same reason: bypasses the orchestrator, tells you nothing about the org-specific routing.
3. DO NOT instantiate a new `BrandingOrchestrator` in the route handler. The production dispatch graph lives on `server.brandingOrchestrator` (decorated in `server.ts:234`). Creating a second instance would have different constructor-injected adapters than the production one and would not exercise the real dispatch.
4. DO NOT hardcode `routedVia: 'embedded'` or `routedVia: 'remote'`. Read it from the orchestrator result: `result.mode`. If the handler hardcodes this, a future bug where the orchestrator's mode dispatch breaks will not be detected by the test button — defeating its purpose.
5. DO NOT skip the mode dispatch by reading `orgs.branding_mode` in the handler and calling the correct adapter directly. Even if the handler ends up invoking the same adapter the orchestrator would have, it bypasses the orchestrator's error-handling contract (degraded result shape, RemoteBrandingMalformedError categorization) and prevents end-to-end exercise of the dispatch path.

**Enforcement:** every test in this plan uses a spy on `server.brandingOrchestrator.matchAndScore` and asserts:
- `expect(spy).toHaveBeenCalledTimes(1)` per request
- `expect(spy).toHaveBeenCalledWith(expect.objectContaining({ orgId, issues: expect.any(Array), guideline: expect.any(Object), ... }))`
- No other branding-service client methods are called (separate spy on `brandingService.listGuidelines` returns 0 calls — asserted in test 1)

**Plan checker: the acceptance criteria for Task 2 include `grep -c "brandingOrchestrator.matchAndScore" packages/dashboard/src/routes/admin/organizations.ts` returns at least 1 AND `grep -c "brandingService.listGuidelines\|/api/v1/health\|/api/v1/guidelines" packages/dashboard/src/routes/admin/organizations.ts` returns 0.**
</pitfall_5_enforcement>

        message: 'Test connection probe — synthetic contrast failure',
        selector: '.probe-btn',
        context:

      // orchestrator, never a shortcut. The test asserts this call count.
      //
      // No try/catch here: the Phase 17 orchestrator contract returns a
      // tagged-union result for all routing failures (kind='degraded').
      // It only throws on programmer errors (missing arguments, broken DI,
      // etc.) — those SHOULD propagate to the Fastify error handler and
      // surface as a 500. Catching would (a) hide real bugs and (b) force
      // us to invent a fake 'unknown' routedVia value that has no contract

      const result = await server.brandingOrchestrator.matchAndScore({

        siteUrl: 'https://test-connection.probe.luqen.local',
        scanId: `branding-test-${randomUUID()}`,

        guideline: syntheticGuideline,

        | {
            ok: true;
            routedVia: 'embedded' | 'remote';
            details: { brandRelatedCount: number; scoreKind: 'scored' | 'unscorable' };
          }
        | {
            ok: true;
            routedVia: 'embedded' | 'remote';
            details: { note: string };
          }
        | {
            ok: false;
            routedVia: 'embedded' | 'remote';
            details: { reason: string; error: string };
          };

      if (result.kind === 'matched') {
        testResult = {
          ok: true,
          routedVia: result.mode,
          details: {
            brandRelatedCount: result.brandRelatedCount,
            scoreKind: result.scoreResult.kind,
          },
        };
      } else if (result.kind === 'degraded') {
        testResult = {
          ok: false,
          routedVia: result.mode,
          details: {
            reason: result.reason,
            // escapeHtml defends against a malicious remote service returning
            // a script-tag-laden error message. Handlebars default-escapes on
            // render, but escaping here too is defense-in-depth in case a
            // future template author switches to triple-brace.
            error: escapeHtml(result.error),
          },
        };
      } else {
        // kind === 'no-guideline'
        testResult = {
          ok: true,
          routedVia: result.mode,
          details: {
            note:
              'Org has no linked guideline; the match layer was not fully exercised. ' +
              'Link a guideline to this org and retry for a complete test.',
          },
        };
      }

      return reply.view('admin/partials/branding-mode-toggle.hbs', { testResult });
    },
  );
