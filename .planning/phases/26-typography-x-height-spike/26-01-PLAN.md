Create `packages/dashboard/scripts/spike-font-metrics.ts` that:

1. Reads `GOOGLE_FONTS_API_KEY` from env (exit with clear message if absent)
2. Defines test families array:
   ```typescript
   const TEST_FAMILIES = [
     'Inter', 'Roboto', 'Open Sans', 'Lato', 'Montserrat',
     'Source Sans 3', 'Noto Sans', 'Playfair Display', 'Merriweather', 'Raleway',
   ];
   ```
3. `resolveTtfUrl(family, apiKey)` function:
   - Calls `https://www.googleapis.com/webfonts/v1/webfonts?key=${apiKey}&family=${encodeURIComponent(family)}`
   - Validates response with a Zod schema: `z.object({ items: z.array(z.object({ family: z.string(), files: z.record(z.string()) })).min(1) })`
   - Returns `item.files['regular'] ?? item.files['400'] ?? Object.values(item.files)[0] ?? null`
4. `extractMetrics(ttfUrl)` function:
   - Fetches TTF as ArrayBuffer: `await fetch(ttfUrl).then(r => r.arrayBuffer())`
   - Parses with `opentype.parse(buffer)` (wrap in try/catch — parse failure = null metrics)
   - Returns `{ unitsPerEm: font.unitsPerEm, os2Version: os2?.version ?? null, sxHeight: (os2?.sxHeight > 0) ? os2.sxHeight : null, sCapHeight: (os2?.sCapHeight > 0) ? os2.sCapHeight : null, xHeightRatio: computed or null }`
5. Main loop: iterate TEST_FAMILIES, call both functions, log structured output per family
6. After loop, compute coverage: `familiesWithXHeight / totalResolved`
7. Print verdict:
   - `>= 0.8` (80%): `VERDICT: VIABLE — ${coverage}% of tested families have sxHeight data`
   - `< 0.8`: `VERDICT: NOT VIABLE — only ${coverage}% of tested families have sxHeight data`

Run the script: `cd packages/dashboard && GOOGLE_FONTS_API_KEY=$KEY npx tsx scripts/spike-font-metrics.ts`

Capture the output in the SUMMARY. The verdict determines whether Plan 02 executes.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard -> Google Fonts API | Outbound HTTP to external API with API key |
| Google Fonts CDN -> opentype.js | External TTF binary parsed locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Information Disclosure | Google Fonts API key | mitigate | Store in env var GOOGLE_FONTS_API_KEY only; never log the key value; use config.ts pattern consistent with other secrets |
| T-26-02 | Tampering | TTF binary from CDN | mitigate | Wrap opentype.parse() in try/catch; treat parse failure as non-fatal metric miss; only fetch from fonts.gstatic.com domain |
| T-26-03 | Spoofing | SSRF via font URL | mitigate | URL always constructed from Google CDN response; never accept user-provided TTF URLs |
| T-26-04 | Denial of Service | Google Fonts API rate limits | accept | Spike fetches only 10 families; production will cache in DB — each family fetched at most once |
</threat_model>
