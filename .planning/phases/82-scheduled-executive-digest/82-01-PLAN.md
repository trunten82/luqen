## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service code → digest_schedules table | All writes go through the repository; channels stored as a controlled JSON string |
| org scope → row visibility | listDigestSchedules filters by orgId; getDue runs system-wide for the scheduler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-01 | Information disclosure | one org reading another org's schedules | mitigate | listDigestSchedules filters by orgId; org_id stored on every row (DEFAULT 'system'); higher layers (Plan 04 routes) pass request.user.currentOrgId |
| T-82-02 | Tampering | SQL injection via schedule fields | mitigate | all repository statements use parameterised better-sqlite3 bindings (@param), never string concatenation of user data |
| T-82-03 | Elevation of privilege | non-admin creating schedules | mitigate | digest.manage permission seeded here; enforced at the route layer in Plan 04 (requirePermission) |
| T-82-04 | Tampering | malformed channels JSON crashing the loader | mitigate | channels written only as JSON.stringify(string[]) from the create path; digestRowToRecord JSON.parse is the single read point |
| T-82-SC | Tampering | npm installs | mitigate | No new dependencies — reuses better-sqlite3 already in the project |
</threat_model>
