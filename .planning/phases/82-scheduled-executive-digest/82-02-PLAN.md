<objective>
Build the digest computation core: a `buildDigest()` service that, for an org-wide or single-site
scope and a time period, computes "what changed" (new vs fixed findings — per-WCAG-criterion deltas
and totals deltas from the stored json_report) and "what's at risk" (the exposure trend: current vs
baseline band + direction, ranked by current band, with deadline countdowns from the existing
legal-exposure driver constants). It exports a typed `DigestData` consumed by every rendering surface
in Plans 03 (PDF/email/scheduler) and 04 (HBS view/API).

Purpose: deliver the DIGEST-02 content logic in conservative framing (band label never a number, no
forbidden words, "no new scan this period" is explicit). One builder, three renderings (D-06).
Output: digest-service.ts, a one-line export of BAND_ORDINAL from legal-exposure.ts, and a service test.

This plan does NOT modify the legal-exposure model — it only EXPORTS the existing BAND_ORDINAL constant
so the trend direction can be computed without re-declaring it.
</objective>
