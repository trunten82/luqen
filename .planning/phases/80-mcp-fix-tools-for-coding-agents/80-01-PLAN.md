<objective>
Extend the existing `executeGenerateFix` LLM capability and its HTTP route so the
fix response carries the WCAG criterion (echoed from input) and a diff/snippet, and
so a `platform: 'wordpress-gutenberg'` input routes through a Gutenberg-block-aware
prompt variant. This is the upstream dependency the dashboard `dashboard_generate_fix`
MCP tool (Plan 03) consumes over HTTP.

Per D-04: REUSE `executeGenerateFix` — do NOT fork a parallel fix engine; keep its
50-pattern hardcoded fallback intact. Per D-05: widen `GenerateFixInput` and
`GenerateFixResult` additively (all new fields optional) so existing call sites
(`llm_generate_fix` MCP tool, dashboard issue-fix UI) keep compiling and behaving.

Purpose: deliver MCPFIX-02 (diff + WCAG criterion) and MCPFIX-04 (WP-block-aware fix
through the same path) at the capability layer.
Output: widened capability types, a Gutenberg prompt builder, an updated HTTP route
response, and tests proving the new behavior.
</objective>
