  <behavior>
    - computeSparklinePoints with gaps: when called with `gaps` Set (indices to skip), the returned points string omits those indices entirely. E.g., values=[80,70,60], gaps=new Set([1]) produces 2 points (index 0 and 2) with x-coordinates still spread across the full width.
    - computeSparklinePoints without gaps: behaves identically to current implementation (backward compatible).
    - computeSparklinePoints with all-gap or fewer than 2 non-gap values returns empty string.
    - New helper `computeTargetY(target: number, values: readonly number[], viewBoxH?: number): number` computes the y-coordinate for a horizontal target line using the same min/max/range logic as sparkline points. Returns the y-coordinate. Needed by Plan 24-02 for the target line.
    - Route: activeSite gains `dimensionSparklines` object with `{ color: { points: string; hasData: boolean }, typography: { points: string; hasData: boolean }, components: { points: string; hasData: boolean } }`. `hasData` is true when dimension has >=2 scored entries. `points` is empty string when hasData is false.
    - Route test: when history has entries where typography is unscorable for 2 of 4 entries, typography sparkline has 2 points (the 2 scored ones) and color/components have 4 points.
    - Route test: when a dimension has only 1 scored entry across history, its hasData is false and points is empty string.
  </behavior>
  <action>
1. **sparkline.ts** — Add optional 4th parameter `gaps?: ReadonlySet<number>` to `computeSparklinePoints`. When provided, filter out indices in the gaps set before computing points. The x-coordinates should be based on the non-gap indices' positions in the original array (so the gap is visible as a spatial break). Implementation: filter `values` to non-gap entries, track original indices for x-positioning. For x calculation: use the original index position `i * step` where step = viewBoxW / (totalValues - 1), so gaps appear as visual breaks in the polyline. Only the y-values and point generation change — gap indices are simply omitted from the points string.

   Also add `computeTargetY(target: number, values: readonly number[], viewBoxH = 40): number` — computes where a horizontal line should sit on the sparkline. Uses the same padding=2, effectiveH, min/max/range logic as `computeSparklinePoints`. Returns `padding + effectiveH - ((target - minVal) / range) * effectiveH`, rounded to 1 decimal. When values is empty or has <2 entries, return -1 (caller should not render).

2. **brand-overview.ts** — After computing `sparklinePoints` for the composite line, extract per-dimension series from the chronological history. For each dimension ('color', 'typography', 'components'):
   - Walk chronological entries. For each entry, check if `entry.result.kind === 'scored'` AND `entry.result[dimension].kind === 'scored'`. If yes, extract the value. If no, mark that index as a gap.
   - Build a `gaps` Set of indices where the dimension is unscorable.
   - Extract `dimensionValues` array (ALL values including placeholders — use the composite overall as placeholder for gap indices, it won't matter since those indices are in the gaps set).
   - Actually simpler: build values array of dimension scores and gaps set simultaneously. Values array has the dimension score at non-gap indices and any number (e.g., 0) at gap indices. Then call `computeSparklinePoints(values, 100, 40, gaps)`.
   - `hasData = (values.length - gaps.size) >= 2`.
   - Add `dimensionSparklines` to `SiteEntry` interface and populate for activeSite.

   The `dimensionSparklines` type:
   ```typescript
   dimensionSparklines: {
     color: { points: string; hasData: boolean };
     typography: { points: string; hasData: boolean };
     components: { points: string; hasData: boolean };
   };
   ```

   Also pass the full chronological `values` array (composite scores) to the view data as `activeSite.compositeValues` — Plan 24-02 will need it for `computeTargetY`.

3. **Tests** — Add sparkline tests for gap-aware behavior and computeTargetY. Add route tests verifying dimension extraction with mixed scorable/unscorable entries.
