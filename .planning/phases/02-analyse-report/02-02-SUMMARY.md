## What Was Built

### LLMClient.analyseReport (llm-client.ts)
New method after `generateFix` section, POSTing to `/api/v1/analyse-report`. Accepts `siteUrl`, `totalIssues`, `issuesList`, `complianceSummary`, `recurringPatterns`, optional `orgId`. Returns `executiveSummary`, `keyFindings`, `patterns`, `priorities`.

### GET /reports/:id/ai-summary (routes/reports.ts)
HTMX partial route with full lifecycle:
1. Auth guard (orgId scoping, admin bypass)
2. Report data load (DB first, filesystem fallback)
3. IssuesList aggregation from `issueGroups` (deduplication by criterion+message key)
4. ComplianceSummary string from `complianceMatrix`
5. RPT-06 pattern detection: `listScans({siteUrl, orgId, status:'completed', limit:5})`, excludes current scan, counts criterion frequency across prior reports, surfaces recurring patterns
6. LLM call via `llmClient.analyseReport()`
7. HTML rendering with `rpt-ai-summary` classes
8. Graceful degradation: `alert--info` notice on LLM error or null reportData/llmClient

`llmEnabled: llmClient !== null` added to all three `reply.view('report-detail.hbs', ...)` calls.

### report-detail.hbs
- AI Summary tab button added after `tab-pages`, guarded by `{{#if llmEnabled}}`
- `panel-ai-summary` div added after `panel-pages` closing tag, with `hx-get`, `hx-trigger="revealed"`, `hx-swap="innerHTML"`, skeleton loading placeholder

### style.css
8 new CSS classes appended after `rpt-fix-hint__actions` section: `rpt-ai-summary`, `rpt-ai-summary__section`, `rpt-ai-summary__heading`, `rpt-ai-summary__body`, `rpt-ai-summary__list`, `rpt-ai-summary__list li`, `rpt-ai-summary__footer`, `rpt-ai-summary__loading`. All use existing design system custom properties.

### en.json (reportDetail section)
8 new i18n keys: `tabAiSummary`, `aiSummaryHeadingExecutive`, `aiSummaryHeadingFindings`, `aiSummaryHeadingPatterns`, `aiSummaryHeadingPriorities`, `aiSummaryUnavailable`, `aiSummaryNone`, `aiSummaryFooter`.
