    1. Boots each Fastify service in-process (reuses Plan 40-01 buildApp factories where possible) and walks `app.printRoutes()` plus the route options to extract `(method, path, requiredPermission, surfaceType)`. surfaceType ∈ {`http-route`, `dashboard-page`, `mcp-tool`}.
    2. For dashboard pages, distinguish from HTTP API routes by URL prefix (`/admin/*`, `/dashboard/*` etc — encode the rule in a constant).
    3. For MCP tools, enumerate from `packages/dashboard/src/mcp/tools/` registrations and read each tool's `requiredScope`/`permission` field.
    4. For OAuth-scoped routes (compliance/branding/llm), record the OAuth scope as the `Permission` value with prefix `oauth:` (e.g. `oauth:compliance:write`).
    5. Build a deduped, sorted-by-permission-then-surface markdown table and write to `docs/reference/rbac-matrix.md`.

    Output file structure:
    ```markdown
    # RBAC Matrix

    _Generated by `npm run docs:rbac` from code. Do not edit by hand._

    _Last generated from commit: <git rev-parse --short HEAD>_

    | Permission | Surface | Surface Type | Source |
    |------------|---------|--------------|--------|
    | reports.export | GET /api/v1/reports/:id/export | http-route | packages/dashboard/src/routes/reports.ts |
    | mcp:dashboard.scan_site | POST /mcp tools/call dashboard_scan_site | mcp-tool | packages/dashboard/src/mcp/tools/scan-site.ts |
    | oauth:compliance:write | POST /api/v1/issues | http-route | packages/compliance/src/routes/issues.ts |
    ...
    ```

    Sort: by Permission ascending, then Surface ascending. Rows MUST be deterministic across runs.

    Add `"docs:rbac": "tsx scripts/generate-rbac-matrix.ts"` to root package.json scripts.
  </action>

          - uses: actions/checkout@v4
