## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Body schema short-circuited auth-gate test paths**
- **Found during:** Task 1 (api-plugins access-control test) and replayed in setup.ts
- **Issue:** Adding `body: <schema>` on routes where preHandler enforces
  permissions causes Fastify to run body validation FIRST. With required body
  fields, an unauthenticated/unauthorised caller gets a 400 "Bad Request" instead
  of the handler's 401/403, breaking access-control tests.
- **Fix:** Dropped `body:` declarations on plugins (install/config) and
  /api/v1/setup. Handlers do their own body validation. Schemas still declare
  params/querystring/response so the route appears in the spec.
- **Files modified:** plugins.ts, setup.ts
- **Commit:** part of `f2daa09` (rectified before merge)

**2. [Rule 1 - Bug] LuqenResponse-wrapped 201 broke setup-api tests**
- **Found during:** Task 1 setup-api regression
- **Issue:** Handler returns `{ message, user }` directly (no LuqenResponse data
  wrapper). Declaring `LuqenResponse(SetupSuccessSchema)` made Fastify
  serialize-strip the actual return shape, producing 500 errors.
- **Fix:** Schema mirrors the bare handler return shape.
- **Commit:** part of `f2daa09`

**3. [Rule 1 - Bug] HtmlPageSchema force-stringified JSON branch on assignments POST**
- **Found during:** Task 2 assignments regression
- **Issue:** The handler returns either JSON or HTML based on Accept header.
  HtmlPageSchema's `response: { 200: Type.String() }` forced the JSON return through
  the string serializer, dropping the response body.
- **Fix:** Drop the response shape on that route — keep only `tags` + params.
  The route still appears in OpenAPI via params declaration.
- **Commit:** part of `af85e9c`

**4. [Rule 1 - Bug] Required querystring fields broke handler-driven 400s**
- **Found during:** Task 1 data-api trends test
- **Issue:** `siteUrl: Type.String()` (required) made Fastify return generic
  "Bad Request" before the handler could emit its specific
  "siteUrl query parameter is required" message.
- **Fix:** All querystring fields kept Optional. Handlers do their own
  required-field checks.
- **Commit:** part of `f2daa09`
