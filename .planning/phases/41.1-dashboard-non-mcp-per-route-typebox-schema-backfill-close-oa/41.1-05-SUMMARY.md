### Task 2 — Schema-fidelity assertion in route-coverage.test.ts

Added a new `it('schema fidelity — ...', ...)` block (148 lines) to the existing `OpenAPI route coverage (dashboard, non-MCP)` describe. Three sub-assertions:

- **A — typed 2xx ≥ 95%.** An operation is "typed" if any 2xx response declares `content`, OR if it is intentionally documented as HTML-rendering (tag `html-page` OR 2xx content includes `text/html`). Measured: **99.28%** of in-scope ops typed (274 of 276).
- **B — every POST/PUT/PATCH declares `requestBody`.** Exemptions: `html-page` routes (HTML rendering, body shape varies) and an explicit `ARCHITECTURALLY_BODYLESS_WRITES` allowlist for the 7 `/api/v1/*` routes that intentionally omit body schemas to preserve auth-gate-before-validation behaviour. Measured: **0** in-scope writes missing body.
- **C — no bare `Default Response` on in-scope ops.** Measured: **0** in-scope ops with bare `Default Response`.

**Out-of-scope filter** (documented in test comment block per CONTEXT.md):
- `/agent/*` — agent.ts route-level Zod migration explicitly deferred (CONTEXT line 161).
- `/graphql`, `/graphiql/*` — framework-injected by GraphQL plugin; schemas live with the plugin.
- `/health`, `/robots.txt` — framework / standard-response endpoints.

**Commit:** `12e2e89`.

## Deviations from Plan

### Auto-fixed Issues (Rule 1-3)

**1. [Rule 3 — Blocking issue] Wave-1 plans missed `tags: ['html-page']` on local HtmlPartialResponse helpers**

- **Found during:** Task 1 metrics check.
- **Issue:** All 17 admin/* files defined a local `HtmlPartialResponse` constant with `produces: ['text/html']` but no tag. After regeneration, 103 POST/PUT/PATCH routes appeared as missing-body without the html-page exemption.
- **Fix:** Added `tags: ['html-page']` to each local helper definition (17 files). Also fixed 6 other local helpers with the same shape (RotateResponse, MixedResponse, ReseedHtmlResponse, MixedHtmlOrJsonResponse) across `oauth-keys.ts`, `roles.ts`, `service-connections.ts`, `system.ts`, `teams.ts`, `system-brand-guidelines.ts`.
- **Files modified:** 23 admin/*.ts files.
- **Commit:** `4660ac9`.

**2. [Rule 3 — Blocking issue] `/admin/audit` and `/admin/audit.csv` had no schema (wave-1 missed `agent-audit.ts`)**

- **Found during:** Task 1 metrics check (bare Default Response list).
- **Issue:** Both routes registered without a `schema:` block — produced bare `Default Response` and were untyped.
- **Fix:** Added `HtmlPageSchema` to `/admin/audit` and a typed CSV schema to `/admin/audit.csv` (text/csv produces, ErrorEnvelope for 4xx). NOT adding `querystring` validation at Fastify level — handler's `safeValidate` normalises empty strings; Fastify-level validation would short-circuit `Test 2: empty outcomeDetail is omitted from filters`.
- **Files modified:** `packages/dashboard/src/routes/admin/agent-audit.ts`.
- **Commit:** `4660ac9`.

**3. [Rule 3 — Blocking issue] Two top-level routes had untyped responses**

- **Found during:** Task 1 metrics check.
- **Issue:** `POST /reports/:id/assignments` (mixed JSON/HTML by Accept) and `POST /session/switch-org` (302 redirects on success, JSON only on errors) lacked typed 2xx responses.
- **Fix:** Added `'html-page'` to the existing tag arrays for both routes (each route's source comment already documented the dual-shape design rationale; this just makes the documentation machine-readable).
- **Files modified:** `packages/dashboard/src/routes/assignments.ts`, `packages/dashboard/src/routes/auth.ts`.
- **Commit:** `4660ac9`.

### Threshold tuning (per plan's escape valve)

The plan permitted "If the 95% target is unreachable due to a small set of routes that legitimately cannot be typed (e.g. plugin-injected catch-alls), add them to the filter with a code comment explaining why." Used twice:

- **Out-of-scope filter** for `/agent/*`, `/graphql`, `/graphiql/*`, `/health`, `/robots.txt` (documented rationale: agent Zod deferral per CONTEXT.md, framework-injected routes).
- **`ARCHITECTURALLY_BODYLESS_WRITES` allowlist** for 7 `/api/v1/*` routes whose source comments already document the no-body decision (Fastify body validation runs before auth preHandler — declaring a body schema would short-circuit the 401/403 path).

The 95% threshold was NOT lowered. Final in-scope ratio is **99.28%** (274/276 typed).
