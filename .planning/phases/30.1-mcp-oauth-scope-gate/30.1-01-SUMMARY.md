# Phase 30.1 Plan 01: MCP OAuth scope-filter gate Summary

Closed the scope-filter bypass discovered during Phase 30 SC#4 walkthrough: OAuth client-credentials tokens with `scope=read` no longer inherit the legacy global `user` role's permissions and therefore land on the empty-permissions branch where `filterToolsByScope` is authoritative. ### Task 2 â€” Rewrite filterToolsByScope + filterResourcesByScope

**Commit:** `4d0ca8a`

**Changes:**
- `packages/core/src/mcp/tool-filter.ts` â€” both `filterToolsByScope` and `filterResourcesByScope` rewritten. New contract:
  - `requiredPermission == null` â†’ always visible (D-04 preserved).
  - `perm === 'admin.system'` â†’ admin-only (explicit gate, no fall-through).
  - `perm` ends with `.create` / `.update` / `.manage` / `.delete` OR equals `admin.users` / `admin.org` â†’ requires `write+`.
  - `perm` ends with `.view` â†’ requires `read+`.
  - Default for unknown permission shapes â†’ fail-safe `write+` (forces explicit classification of new permissions).
- `packages/core/src/mcp/__tests__/tool-filter.test.ts`:
