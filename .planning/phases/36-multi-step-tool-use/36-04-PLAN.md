<interfaces>
SSE frame contract from 36-02:
```ts
{ type: 'tool_started', toolCallId: string, toolName: string }
{ type: 'tool_completed', toolCallId: string, toolName: string,
  status: 'success' | 'error', errorMessage?: string }
```

Special overload from 36-03: when iteration cap is hit, AgentService emits
`{ type: 'tool_completed', toolCallId: '__loop__', toolName: '__loop__',
   status: 'error', errorMessage: 'iteration_cap' }` — frontend treats this
as the cap-hit chip (different label + class), not a real tool error.

Existing client conventions (agent.js header lines 6–9 + line 396):
- DOM mutation via createElement + textContent — never innerHTML for untrusted text.
- EventSource is created in `connectStream()` (search for `new EventSource` in agent.js).
- BEM prefix locked to `agent-drawer__*` — chip strip continues this convention.
</interfaces>
