<objective>
Server-side core of the phase. Three jobs:
1. New POST `/agent/active-org` endpoint — validates admin.system, validates orgId is in `listOrgs()`, writes `active_org_id`, audits, returns the new active org for inline UI feedback.

Purpose: Server-side security boundary (AORG-04) and per-turn rebind (AORG-02). Plans 38-01 (storage) and 38-02 (UI) feed into it.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /agent/active-org | Authenticated, CSRF-protected. |
| POST handler → user-repository | Trusted internal call after permission check. |
| route → AgentService.runTurn | Internal; orgId derived by resolver, never trusted from request body. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-06 | Spoofing | POST /agent/active-org | mitigate | JWT preHandler verifies caller identity; userId comes from token, never request body. |
| T-38-07 | Tampering | POST body orgId | mitigate | Validated against `listOrgs()` allowlist; unknown orgs → 400 + audit. |
| T-38-08 | Repudiation | switch action | mitigate | Every call (success or denied) emits an audit row with toolName='org_switched' + from/to orgs. |
| T-38-09 | Information disclosure | cross-org GET conversation | mitigate | Allowlist check: `permissions.has('admin.system')` is the ONLY way to bypass org scope. Non-admins remain scoped. |
| T-38-10 | DoS | rapid POSTs to /agent/active-org | accept | Existing global rate-limit covers this; org list query is small (3 orgs). |
| T-38-11 | Elevation of privilege | non-admin forging POST | mitigate | 403 + audit row with outcomeDetail='not_admin_system'; verified by test. |
</threat_model>
