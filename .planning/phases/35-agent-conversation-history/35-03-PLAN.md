<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Fastify route | Untrusted JSON body + query params cross here; JWT preHandler attaches `request.user` and permissions |
| Route → Repository | Route passes only the resolved `orgId` (from JWT), never client-supplied orgId |
| Route → Audit log | Delete/rename writes cross this boundary; must be executed server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-08 | S (Spoofing) | All 5 handlers | mitigate | `request.user === undefined → 401`; `resolveAgentOrgId` re-derives org from JWT; client cannot inject orgId |
| T-35-09 | T (Tampering) | POST rename/delete | mitigate | @fastify/csrf-protection (global, `server.ts:366`) enforces x-csrf-token; zod body validation on rename; id from URL param always combined with orgId in repo |
| T-35-10 | I (Info disclosure) | GET search/get/list | mitigate | Repo enforces `user_id + org_id + is_deleted = 0`; route never exposes audit flags to user-facing payload |
| T-35-11 | R (Repudiation) | POST delete, POST rename | mitigate | `storage.agentAudit.record(...)` writes per action with userId + orgId + conversationId + meta |
| T-35-12 | D (DoS) | GET search, GET list | mitigate | zod caps `limit ≤ 50`, `q ≤ 200 chars`; rate-limit inherited from existing agent scope |
| T-35-13 | E (Elevation) | POST delete | mitigate | `softDeleteConversation` returns false when orgId mismatch → 404; no privilege inference from URL path |
</threat_model>
