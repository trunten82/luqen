# Phase 35 — UI Design Contract

> Visual and interaction contract for the agent drawer's stacked history panel. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (server-rendered Handlebars + HTMX 2.0 + vanilla JS) |
| Preset | not applicable |
| Component library | none — reuse existing `packages/dashboard/src/static/style.css` tokens and BEM classes |
| Icon library | inline SVG (stroke-based, `currentColor`, `aria-hidden="true"`) — matches existing drawer icons |
| Font | `var(--font-sans)` — system-ui, -apple-system, 'Segoe UI', Roboto, Helvetica, Arial, sans-serif |

**Stack posture (locked by project conventions):**
- CSP-strict: no inline handlers, no `eval`; `data-action="..."` attributes wire into `agent.js` delegated listener.
- All visible strings via `{{t "key"}}` → `packages/dashboard/src/i18n/locales/en.json`. Never hardcode English.
- Design tokens from `style.css` only — do NOT invent new CSS custom properties or base classes.
- BEM prefix `agent-drawer__*` for new drawer-scoped classes.
- Cross-section swaps use plain `fetch()` (not `hx-select`) per HTMX 2.0 caveat.
- Never nest `<form>` inside `<table>` cells — use `hx-post` on buttons + CSRF meta-tag interceptor.

## Spacing Scale

Declared values (multiples of 4, all sourced from existing `--space-*` tokens):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`--space-xs`) | Gap between entry title and meta line |
| sm | 8px (`--space-sm`) | Menu item padding, menu gap, search clear-button inset |
| md | 16px (`--space-md`) | Entry vertical padding, panel header padding, search bar vertical padding |
| lg | 24px (`--space-lg`) | Empty-state block padding |
| xl | 32px (`--space-xl`) | (unused this phase) |
| 2xl | 48px (`--space-2xl`) | Empty-state top margin |

Exceptions:
- Three-dot menu trigger is a **32×32px** touch target (below the 44px WCAG AAA advisory) because it lives inside a dense list row and the whole row is also keyboard-activatable via Enter → Resume; the secondary action must not dominate the row. Contrast and focus ring remain WCAG 2.2 AA compliant (see Typography/Color sections). Screen-reader users reach Rename/Delete from the row context menu equivalent, not the 32px trigger alone.

## Typography

All sizes use existing `--font-size-*` tokens. Line-height uses `--leading-*` tokens.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Body | 14px (`--font-size-sm`) | 400 | 1.5 (`--leading-normal`) | Entry title, search input, menu items, snippet text |
| Label | 12px (`--font-size-xs`) | 400 | 1.5 (`--leading-normal`) | Entry meta (timestamp + message count), empty-state body |
| Heading | 18px (`--font-size-lg`) | 600 | 1.2 (`--leading-tight`) | Panel header title ("History"), empty-state heading |
| Display | — | — | — | not used this phase |

Weights declared: **400 (regular)** + **600 (semibold)**. No other weights permitted.

Truncation:
- Entry title: single line, `text-overflow: ellipsis`, `overflow: hidden`, `white-space: nowrap`. Full title exposed via `title` attribute for cursor hover and accessible name.
- Snippet (search result): two-line clamp via `-webkit-line-clamp: 2` with fallback `max-height` computed from 2 × line-height.

## Color

Tokens only. 60/30/10 split honours the existing Luqen palette (green accent).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-primary)` — `#ffffff` light / `#1f2937` dark | Stacked panel background, search input background |
| Secondary (30%) | `var(--bg-secondary)` — `#fafafa` light / `#111827` dark | Hovered/focused entry row, menu popover background, delete-confirm row background (via `card--muted`) |
| Accent (10%) | `var(--accent)` — `#15803d` light / `#4ade80` dark | **Reserved for:** (1) focus ring on interactive elements, (2) `<mark>` highlight background for matched search terms, (3) primary "Resume" affordance on the currently-focused entry (underline colour only), (4) back button chevron on hover. Nothing else. |
| Destructive | `var(--status-error)` + `var(--status-error-bg)` — `#b91c1c` / `#fee2e2` | Delete-confirm row's Delete button only (`btn btn--danger`). Trash icon in the three-dot menu inherits `--status-error` on hover. |

**Accent reserved-for list (explicit, final):**
1. Focus-visible outline on list items, menu items, search input, back button, three-dot trigger
2. `<mark>` background for the matched search term inside snippet rows
3. Underline colour of the active/focused entry row
4. Back-button hover chevron stroke

**Accent NOT used for:** all-row hover backgrounds, entry borders, active conversation marker (use a left-border in `--border-color-strong` instead), timestamp colour, or any decorative accent.

Contrast gates (must hold in light + dark):
- Entry title on dominant bg ≥ 4.5:1 (`--text-primary` on `--bg-primary` — already WCAG AA in tokens)
- Meta label on dominant bg ≥ 4.5:1 (`--text-secondary` on `--bg-primary`)
- `<mark>` background (`--accent-light`) with `--text-primary` foreground ≥ 4.5:1 — verify in executor phase; fallback to `--status-info-bg` if accent-light fails
- Focus outline: reuse `--focus-outline` (3px `--accent`, offset 2px) — token-compliant

## Interaction States

| Element | Default | Hover | Focus-visible | Active/Selected | Disabled |
|---------|---------|-------|---------------|-----------------|----------|
| List entry row | `bg-primary`, no border | `bg-secondary` | 3px `--accent` outline, offset 2px | left-border 3px `--border-color-strong` when it matches the currently-open conversation | N/A |
| Three-dot trigger | transparent, `--text-muted` icon | `bg-tertiary` | 3px `--accent` outline | `aria-expanded="true"` + `bg-tertiary` | N/A |
| Menu item | `bg-secondary`, `--text-primary` | `bg-tertiary` | `bg-tertiary` + 2px `--accent` inset outline | N/A | `--text-muted`, `cursor: not-allowed` |
| Menu item Delete | `--status-error` text | `--status-error-bg` | same + outline | N/A | N/A |
| Search input | `bg-primary`, `--border-color` | `--border-color-strong` | `--focus-outline` | filled: clear-button visible | N/A |
| Rename `<input>` | `bg-primary`, `--border-color` | `--border-color-strong` | `--focus-outline` | validation error: `--status-error` border + `form-hint` message | N/A |
| Back button | `btn btn--ghost btn--sm` tokens | existing | existing | N/A | N/A |

Motion:
- Panel slide-in: `transform: translateX(0)` from `translateX(100%)`, duration `var(--transition-base)` (250 ms). Honours `prefers-reduced-motion: reduce` → no transition.
- Menu popover: fade-in 150 ms (`var(--transition-fast)`). Reduced motion → instant.
- Delete-confirm row: no animation (state swap is instantaneous to make the action feel responsive and reversible).

## Keyboard & Screen-Reader Contract (AHIST-05, WCAG 2.2 AA — locked requirement)

| Context | Key | Effect |
|---------|-----|--------|
| Messages view, drawer open | History button focus + Enter/Space | Opens stacked panel; focus moves to Back button |
| Stacked panel | Esc | Closes history → returns to messages, focus returns to History trigger in header |
| List | ArrowDown / ArrowUp | Moves roving `tabindex` through entries (roving focus pattern) |
| List entry focused | Enter | Resumes conversation (fetches full history, swaps back to messages view) |
| List entry focused | Shift+F10 / Context-menu key | Opens three-dot menu (same effect as clicking the trigger) |
| Three-dot menu open | ArrowDown / ArrowUp | Moves through menu items |
| Three-dot menu open | Esc | Closes menu, focus returns to trigger |
| Three-dot menu open | Enter on Rename | Replaces row with inline rename input; input is auto-focused and text selected |
| Rename inline | Enter | Saves; Esc | cancels, restores entry |
| Three-dot menu open | Enter on Delete | Replaces row with confirm row; **focus moves to Cancel** (safer default) |
| Delete confirm row | Tab/Shift+Tab | Cycles between Cancel and Delete; Esc | cancels, restores entry |
| Search input focused | Esc | Clears input (if non-empty); second Esc closes panel |

ARIA:
- Stacked panel: `role="region" aria-label="{{t 'agent.history.panelLabel'}}"`, `aria-hidden` toggles with visibility.
- List: `<ul role="list">` — entries are `<li>` with `role="button" tabindex` roving pattern (0 for focused item, -1 for rest).
- Search input: `<input type="search" role="searchbox" aria-controls="agent-history-list" aria-describedby="agent-history-search-hint">`.
- Search results count live region: `<div role="status" aria-live="polite" class="sr-only">` announces "{{t 'agent.history.resultsCount' count=N}}" after debounce settle.
- Three-dot trigger: `aria-label="{{t 'agent.history.item.menuLabel' title=entry.title}}" aria-haspopup="menu" aria-expanded`.
- Menu popover: `role="menu"`; items `role="menuitem"`.
- Delete confirm row: `role="alertdialog" aria-labelledby="..."`, containing two buttons.
- `<mark>` elements inside snippet must NOT rely on colour alone — they carry a `font-weight: 600` boost so colour-blind and high-contrast-mode users see the highlight.

Focus lands on Cancel (safer default). No modal, no undo toast (per D-05a). Soft-delete is one-way from the user's perspective — confirm row copy does not promise recovery.

## States — the seven every surface must handle

| State | Visual |
|-------|--------|
| Empty (new user, zero conversations) | Centred icon + heading "No conversations yet" + body copy. No list, no sentinel. |
| Loading (first paint, initial page fetch) | 5 skeleton rows (title bar + meta bar) with `--bg-tertiary` pulse (respects reduced motion → static). No spinner. |
| Loading (subsequent page at sentinel) | 1 skeleton row below last entry. Sentinel hides during fetch. |
| Populated | List of entries, newest first, sentinel at bottom. |
| Search active, results present | List filters to matches; each row shows highlighted snippet in place of default meta. SR live region announces count. |
| Search active, zero results | Empty state variant with query term echoed back. |
| Error (list fetch failed) | Inline error block with heading, body, Retry button. Replaces list body, not the whole panel (search + back button stay usable). |
| Error (pagination fetch failed) | Inline error row at sentinel position with Retry link; previously-loaded entries remain visible. |
| Error (rename failed) | `form-hint--error` under the rename input; input retains value and focus for re-submission. |
| Error (delete failed) | Confirm row stays; red `form-hint` below Cancel/Delete buttons with retry copy. |
