       - Body schema (zod): { content: string trim min 1 max 8000 }

       - Loads messages (excluding superseded). Validates that :mid is the most-recent USER message — else 400 'not_most_recent_user'.

       - Audit: toolName='share_created', argsJson={shareId, anchorMessageId: mid}.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /retry, /edit-resend, /share | CSRF + auth + org-resolve enforced; zod validates body/path |
| anonymous URL → GET /agent/share/:id | Auth still required; org-membership enforced AFTER token resolution |
| AbortSignal → AgentService loop | Internal; no external untrusted input flows here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-09 | Spoofing | /retry, /edit-resend, /share | mitigate | Auth guard on every handler; resolveAgentOrgId derives orgId from JWT, never trusted from request body |
| T-37-10 | Tampering | edit-resend body | mitigate | zod EditResendBodySchema enforces 1-8000 char trimmed string; rejects type confusion |
| T-37-11 | Repudiation | share link creation | mitigate | agentAudit.append with toolName='share_created', argsJson includes shareId, userId from session |
| T-37-12 | Information disclosure | GET /agent/share/:id | mitigate | Three gates: (a) auth required, (b) link.orgId === session.orgId, (c) conversation org-guarded read. CONTEXT explicitly forbids public access. |
| T-37-13 | Denial of service | /share creation flood | mitigate | Rate-limit inherited from /agent scope (60/min/user); per-conversation share count is naturally low |
| T-37-14 | Elevation of privilege | edit-resend on a non-owned conversation | mitigate | Repo's getConversation enforces org_id WHERE clause; foreign-org → 404 not 403 (no leak of existence) |
| T-37-15 | Tampering | retry on superseded message | mitigate | markMessagesSuperseded with idempotent WHERE status IN ('streaming','final','stopped'); already-superseded rows yield 0 rows affected → 404 |
</threat_model>
