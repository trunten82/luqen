<decisions>
## Implementation Decisions

### agent.js split shape
- **4 modules per TRIAGE:** `agent-history.js` (panel + search + cards + load + soft-delete), `agent-tools.js` (chip strip + tool_started/tool_completed SSE handlers + i18n bridge), `agent-actions.js` (retry + copy + share + edit-resend + edit-form handlers + clipboard helpers + ClipboardItem(Promise)), `agent-org.js` (switcher change handler + autoSwitchOrgIfNeeded + initOrgSelectPreviousOrgId).
- `agent.js` remains the entry — keeps drawer toggle, openStream, ensureAssistantBubble, init() bootstrap, document-level click delegation. Module-specific delegation moves into the respective modules' own document-level listeners (each in its own IIFE).
- Each module starts with `;(function () { ... })();` IIFE, no shared globals beyond `window` references.
- Shared utilities (csrfToken, byId, announce, getConversationId, setConversationId, formatToolI18n, readToolI18n, renderMarkdownInto, recordMarkdownSource, readMarkdownSource, writeToClipboard) are exposed on `window.__luqenAgent` namespace from agent.js so the modules can consume without circular imports.
- Target LOC after split: agent.js ~700, agent-history.js ~500, agent-tools.js ~150, agent-actions.js ~600, agent-org.js ~200.

### Module loading
- **Multiple `<script>` tags in `agent-drawer.hbs`**, in dependency order: agent.js first (defines `__luqenAgent` namespace), then agent-history.js, agent-tools.js, agent-actions.js, agent-org.js. All `defer` so init() runs after DOM ready.
- No bundler. Matches existing pattern (agent-audit.js + agent.js are separate today).
- `dist/static/` build script copies all `.js` files automatically — no build-script changes needed.

### MCP scope-filter test fixes (DI-32-01..03)
- **Update test scope expectations**, not production code. Tests at `packages/dashboard/tests/mcp/{data-tools,admin-tools,http}.test.ts` currently pass `scopes: ['read']` to tool-dispatch helpers; current production rules require `write` (or `admin`) for the failing tools.
- For each failure: change the test's scope array to include the tier the production rule requires. Add a comment pointing to the rule definition (e.g. `mcp/middleware.ts` or wherever scope tiers are enforced).
- Production scope-filter logic stays untouched — it's enforcing the correct security boundary.

### Auth-flow returnTo (DI-31.2-01)
- Investigate the original deferred-items.md entry under `milestones/v3.0.0-phases/31.2-mcp-access-control-refinement/deferred-items.md`. Apply the smallest fix that closes the deferred item with a passing test.

### Migration 059 column assertion (DI-38-01)
- **Promote the CI patch to permanent.** The CI fix added `expires_at` to the expected columns in `migration-058-059.test.ts`. Verify the assertion is correct against migration 059 (no `expires_at`) AND migration 060 (which adds `expires_at`). If the test should assert columns AT migration 059 specifically, restructure the test to clarify which migration's state is being checked.

### Multi-step e2e fetch loader (DI-37-01)
- **Promote the CI patch to permanent.** The CI fix stubs `fetch` in beforeEach because agent.js calls `loadPanel()` on stream done. Confirm that pattern is the right shape for the test (vs adding fetch to the JSDOM harness defaults). Add a comment referencing Phase 37 cross-cutting if not already present.

### shareAssistant test mock (DI-38-02b)
- The test (`tests/static/agent-actions-handlers.test.ts:387-400`) was written against `writeToClipboard(url)` directly. After the ClipboardItem(Promise) change in Phase 37, the share path went through `navigator.clipboard.write([new ClipboardItem(...)])` for browsers that support it, falling back to `writeToClipboard` in JSDOM. The CI fix already added the writeText fallback so the test works. Verify the test exercises both ClipboardItem and writeText paths cleanly, OR split into two tests if one path is silently dead.

<specifics>
## Specific Ideas

- After agent.js split lands, `tests/e2e/agent-panel.test.ts` LOC ceiling drops from 2300 → ~1000 (agent.js stub size). This becomes part of 39.1-02's acceptance criteria.
- New module test files should follow the same JSDOM harness pattern as `tests/static/agent-actions-handlers.test.ts` and `tests/static/agent-org-switch.test.ts`.
- The `__luqenAgent` namespace is a one-way export: agent.js fills it, modules read it. Modules don't write back to the namespace. Avoids order-of-load coupling.
- 39.1-01 plan should run all 6 test fixes in one commit per item (TDD: red → green per item) to keep blame easy.
</specifics>
