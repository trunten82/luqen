# Phase 39.1 Plan 01: Deferred-Item Test-Staleness Fixes Summary

Bundled 5 test-only test-staleness fixes covering 7 deferred items (DI-31.2-01,
DI-32-01..03, DI-37-01, DI-38-01, DI-38-02b) into a single TDD pass with one
commit per DI scope. Production code is untouched; full dashboard suite green.

## Outcomes

- **DI-31.2-01** — `tests/e2e/auth-flow-e2e.test.ts` returnTo assertions now
  use `toMatch(/^\/login\?returnTo=%2Fhome(?:&.*)?$/)` so future query-string
  additions don't re-break the test. Both cases (unauth GET /home, post-logout)
  pass. **Commit:** `a8830c4`.

- **DI-32-01..03** — `tests/mcp/data-tools.test.ts` `scopes:['read','write']`
  fixtures now carry inline provenance comments pointing at
  `src/mcp/middleware.ts` scope-filter rule. `admin-tools.test.ts` and
  `http.test.ts` already carried explanatory comments from earlier work
  (commit `29d5faa`). All 6 previously-failing MCP scope cases pass; 38 MCP
  cases total. **Commit:** `919e02b`.

- **DI-37-01** — `tests/e2e/agent-multi-step.e2e.test.ts` harness was
  refactored to thread `fetch` through the IIFE call signature as the 4th
  argument, matching the canonical pattern in
  `tests/e2e/agent-history.e2e.test.ts:262`. The earlier temporary
  `beforeEach(stubFetchOk)` patch is removed in favour of a permanent
  threaded loader. All E1..E4 cases pass. **Commit:** `564d7f8`.

- **DI-38-01** — `tests/db/migration-058-059.test.ts` renamed the
  agent_share_links column-list `it()` to "creates agent_share_links table
  with the expected post-migration-060 columns" and added an inline comment
  explaining that `MigrationRunner.run(DASHBOARD_MIGRATIONS)` runs ALL
  registered migrations (so `expires_at`, added by migration 060, is
  expected). 11 cases pass. **Commit:** `e09fc9f`.

- **DI-38-02b** — `tests/static/agent-actions-handlers.test.ts` Test 12 split
  into 12a (ClipboardItem(Promise) + `clipboard.write` happy path) and 12b
  (writeText fallback when ClipboardItem unavailable). 12a installs a
  FakeClipboardItem on both `win.ClipboardItem` and `globalThis.ClipboardItem`
  so the production bare-identifier `new ClipboardItem(...)` call resolves to
  the test double; `teardownHarness` clears the global to prevent leakage to
  12b. 22 cases pass; 54 cases pass across `tests/static/`. **Commit:**
  `970d831`.

### Auto-fixed issues

**1. [Rule 1 - Bug] ClipboardItem bare-identifier resolution in jsdom realm**

- **Found during:** Task 5 first run.
- **Issue:** Setting only `h.win.ClipboardItem = FakeClipboardItem` was not
  enough to make the production `new ClipboardItem(...)` call (bare
  identifier inside `agent.js` line ~1860) resolve to the test double — the
  `clipboardWriteStub` was never invoked.
- **Fix:** Also set `globalThis.ClipboardItem = FakeClipboardItem` because
  the function loaded via `new win.Function(...)` falls through to
  globalThis for free identifiers in the jsdom realm. Added a
  `delete globalThis.ClipboardItem` in `teardownHarness` to prevent leakage
  to Test 12b (which asserts `ClipboardItem` is undefined to validate the
  fallback path).
- **Files modified:** `tests/static/agent-actions-handlers.test.ts`.
- **Commit:** `970d831`.
