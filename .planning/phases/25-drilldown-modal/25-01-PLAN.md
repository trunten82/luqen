<objective>
Add a drilldown modal to the report detail brand score panel. Clicking a scored sub-score
dimension row (color, typography, or components) fires an HTMX GET that loads a modal
showing the individual failing elements for that dimension, filtered from the scan's
stored JSON report.

Purpose: Users can see exactly WHICH elements are failing and WHY for each brand score
dimension, turning the summary score into actionable remediation guidance (BDRILL-01..03).

Output: Drilldown service, API endpoint, modal template, clickable panel rows, tests.
</objective>

  <action>
    **1. Create `packages/dashboard/src/services/brand-drilldown.ts`:**

    Export a `DrilldownDimension` type: `'color' | 'typography' | 'components'`.

    Export a `DrilldownIssue` interface:
    ```typescript
    interface DrilldownIssue {
      readonly selector: string;
      readonly context: string;
      readonly message: string;
      readonly code: string;
      readonly matchDetail: string;
      readonly strategy: string;
    }
    ```

    Export `filterDrilldownIssues(dimension: DrilldownDimension, reportData: unknown): readonly DrilldownIssue[]`.

    Implementation:
    - Extract `pages` array from reportData (defensive: if missing, return []).
    - Flatten all issues across all pages.
    - Filter by dimension:
      - **color**: issue.code includes any of `['Guideline1_4.1_4_3', 'Guideline1_4.1_4_6', 'Guideline1_4.1_4_11']` AND issue.brandMatch?.matched === true.
      - **typography**: issue.brandMatch?.matched === true AND issue.brandMatch?.strategy === 'font'.
      - **components**: issue.brandMatch?.matched === true AND issue.brandMatch?.strategy === 'selector'.
    - Map matching issues to DrilldownIssue shape.
    - Deduplicate by `code + selector` (same element + same rule = one row). Keep first occurrence.
    - Truncate context to 200 chars max (HTML snippets can be large).
    - Return frozen array.

    Also export `isValidDimension(s: string): s is DrilldownDimension` guard function.

    **2. Add GET /reports/:id/brand-drilldown route to `packages/dashboard/src/routes/reports.ts`:**

    Add the route AFTER the existing `GET /reports/:id/ai-summary` handler. The route:
    - Reads `dimension` from query string. If missing or invalid (not color/typography/components), return 400 with empty HTML.
    - Loads scan via `storage.scans.getScan(id)`. If null or not completed, return 404.
    - Applies org-scoping check (same pattern as the existing report detail route: admin sees all, others check orgId match).
    - Loads report data via `storage.scans.getReport(id)` + `normalizeReportData`.
    - Calls `filterDrilldownIssues(dimension, reportData)`.
    - Loads `brandScore` via `storage.brandScores.getLatestForScan(id)` to get the sub-score value for the header.
    - Renders `partials/brand-drilldown-modal.hbs` with `{ issues, dimension, dimensionLabel, dimensionScore }`.
    - `dimensionLabel` is the i18n key: `t('reportDetail.brandScoreColorContrast')` / `t('reportDetail.brandScoreTypography')` / `t('reportDetail.brandScoreComponents')` based on dimension.
    - `dimensionScore` is extracted from brandScore[dimension].value (if scored) or null.
    - Returns HTML content-type (same pattern as fix-suggestion and ai-summary routes).

    **3. Write tests:**

    `tests/services/brand-drilldown.test.ts`: Unit tests for filterDrilldownIssues with fixture data covering all behaviors listed above.

    `tests/routes/brand-drilldown-route.test.ts`: Integration test using the same Fastify test server pattern from `tests/routes/reports.test.ts`. Test: valid dimension returns 200 HTML, invalid dimension returns 400, missing scan returns 404. Use a minimal json_report fixture with branded issues.
