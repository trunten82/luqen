<gate_math>
**PASS condition (both must hold):**
1. `post_grand_median_ms <= baseline_grand_median_ms * 1.15`
2. For every site s: `post_per_site_median_ms[s] <= baseline_per_site_median_ms[s] * 1.25`

**FAIL condition (either triggers):**
1. `post_grand_median_ms > baseline_grand_median_ms * 1.15` — grand regression
2. Any site s where `post_per_site_median_ms[s] > baseline_per_site_median_ms[s] * 1.25` — per-site regression, even if grand median is OK

**Sentinel handling:** If any site has `-1` in EITHER baseline OR post (meaning all 3 measured runs failed for that site), the gate computation EXCLUDES that site from BOTH sides of the comparison AND flags it explicitly in the verdict file. Do NOT silently drop a site; surface it to the human.

**Example:**
```
baseline grand = 8200 ms
post grand     = 9100 ms
delta          = +11.0% → PASS (< 15%)

baseline per-site = { aperol: 4500, sap: 12000, inps: 6500, sky: 9500 }
post per-site     = { aperol: 4800, sap: 13200, inps: 8500, sky: 10000 }
per-site deltas   = { aperol: +6.7%, sap: +10.0%, inps: +30.8%, sky: +5.3% }
                                                   ^^^^^^^^^^^ FAIL (> 25%)
```
In the above example, grand median is under 15% but inps regressed more than 25% → FAIL.
</gate_math>
