## Outstanding gaps (deferred to Phase 31.2)

Smoke also surfaced two architectural gaps that require more than an inline fix:

- **G1 — Org-scoped DCR client revocation.** `/admin/clients` is currently gated by `admin.system` only. Org admins should be able to revoke DCR clients registered within their own org without requiring a global admin. Needs RBAC extension + UI changes.
- **G10 — `mcp.use` RBAC permission.** Any dashboard user can open an MCP connection today; there's no permission gate on "may use MCP". User requested: a specific org-scoped permission enabling MCP access, with tool visibility driven by the user's real RBAC rather than broad OAuth scope bundles.

Both routed to the inserted Phase 31.2 — MCP Access Control Refinement. Also carrying G9 (service-side WWW-Authenticate parity for compliance/branding/llm) into that phase.
