## Decisions Made

- **mcp.use group = 'Administration'** — keeps the `/admin/roles` checkbox layout compact and aligns with the existing admin.* siblings; no new UI section required.
- **No default grant in ORG_*_PERMISSIONS** — the per-role templates drive new-org role creation. Adding mcp.use there would re-introduce the default-on behaviour that D-04's second sentence forbids. Migration 054 back-fills ONLY existing rows.
- **migration-055 dropped (D-17 discovery)** — `oauth_clients_v2.registered_by_user_id` column + FK `dashboard_users(id) ON DELETE SET NULL` + index `idx_oauth_clients_v2_user_created` already exist in migration 049 (lines 1290-1296 of migrations.ts, landed by Phase 31.1). Plan 01 ships ONLY migration 054.
- **findByOrg does NOT filter on revoked_at** — Plan 04 switches `repo.revoke()` from DELETE to UPDATE revoked_at; the admin UI needs revoked rows visible to render the Revoked badge (D-24). Test 6 locks this invariant so future refactors can't accidentally hide revoked rows.
- **findByOrg('system') short-circuits to []** — admin.system uses listAll(); `system` is not a real team scope. Preferred over silently returning all-orgs because a buggy caller passing 'system' would otherwise drop cross-org isolation.
