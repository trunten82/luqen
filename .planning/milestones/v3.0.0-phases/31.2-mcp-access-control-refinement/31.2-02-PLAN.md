<objective>
Wire the `mcp.use` RBAC gate into the OAuth 2.1 Authorization Code flow, simplify scope handling (admin.* retired at the OAuth layer), retire the dashboard-side `client_credentials` grant, backfill `oauth_clients_v2.registered_by_user_id` at first consent, implement the `/session/switch-org` route that backs the D-05 switch-org CTA, and thread `clientId` through the signer so Plan 04 can enforce a post-JWT revoked-client check.

Purpose: Block unauthorized MCP connections at the exact moment a user tries to authorise a client (not later at tool-list time); provide the switch-org route the consent screen redirects to (D-05 — primary UX of Phase 31.2 goal clause a); delete a deprecated code branch (admin.* scopes); establish the user ↔ DCR client linkage that Plan 04 needs to scope /admin/clients by org; lay the signer-side groundwork so Plan 04's verifier-side client-status check (D-20 bullet 3) can land cleanly.
Output: tightened authorize.ts + token.ts + well-known.ts + consent view; new /session/switch-org handler + tests; signer interface extended with clientId; comprehensive route tests.
</objective>
