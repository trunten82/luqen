## Decisions Made

- **D-23 — shared factory NOT extracted.** Kept three per-package factories. Rationale: API-key env-var names differ (COMPLIANCE_API_KEY / BRANDING_API_KEY / LLM_API_KEY) and per-package TokenPayload types diverge subtly. Extraction needs a generic or type-erasure step that buys little over 15 lines of per-service duplication. Matches the 31.1 D-34 call on `createJwksTokenVerifier`. A later housekeeping phase can revisit both together.
- **Public-URL defaults.** The plan's example text mentioned ports 4100/4200/4300 for compliance/branding/llm. Actual well-known routes declare 4000/4100/5100. Since must_have #7 requires "matches the existing well-known route's self-declared resource URL", I used the real port numbers — otherwise clients would receive one URL from `/.well-known/oauth-protected-resource` and a different one from the 401 header, defeating the purpose.
- **Required option (not optional).** `resourceMetadataUrl` is REQUIRED. This forces every call site to supply a URL at compile time — no silent fallback to `undefined` can regress the header. TypeScript surfaced all three server.ts call sites; no other callers exist.
