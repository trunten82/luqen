    ```typescript
    export function filterToolsByScope(
      allTools: readonly ToolMetadata[],
      tokenScopes: readonly string[],
    ): readonly string[] {
      const hasAdmin = tokenScopes.includes('admin');
      const hasWrite = tokenScopes.includes('write') || hasAdmin;
      const hasRead = tokenScopes.includes('read') || hasWrite;

      return allTools
        .filter((t) => {
          if (t.requiredPermission == null) return true; // D-04: unannotated always visible
          if (hasAdmin) return true;                     // admin scope: all tools
          const perm = t.requiredPermission;

          // Admin-only permission: never granted below `admin` scope.
          if (perm === 'admin.system') return false;

          // Write-tier permissions: create / update / manage / admin.users / admin.org.
          if (
            perm.endsWith('.create') ||
            perm.endsWith('.update') ||
            perm.endsWith('.manage') ||
            perm.endsWith('.delete') ||
            perm === 'admin.users' ||
            perm === 'admin.org'
          ) {
            return hasWrite;
          }

          // Read-tier: `.view` and any other read-shaped permission.
          if (perm.endsWith('.view')) return hasRead;

          // Default: unknown permission shape → treat as write-tier (fail safe —
          // a new permission MUST be explicitly classified, not fall through to
          // the most permissive tier).
          return hasWrite;
        })
        .map((t) => t.name);
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External MCP client → dashboard `/api/v1/mcp` | Untrusted bearer token crosses here; only the signature/scope claim is trusted. The `sub` is an OAuth clientId, NOT a dashboard user. |
| dashboard MCP auth preHandler → role-repository | Internal call; feeds `sub` into `getUserPermissions` which CURRENTLY leaks user-role perms to service callers. |
| @luqen/core filter → dashboard tool metadata | Internal; filter consults `ToolMetadata.requiredPermission` declared in `packages/dashboard/src/mcp/metadata.ts` + `tools/admin.ts`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30.1-01 | Elevation of privilege | role-repository.ts `getUserPermissions` (unknown-sub branch) | mitigate | Task 1: return empty Set instead of legacy `user`-role fallback. Removes the path where a service-to-service caller inherits end-user permissions by accident. Locked repository tests prevent regression. |
| T-30.1-02 | Elevation of privilege | filterToolsByScope + filterResourcesByScope (scope-tier rules) | mitigate | Task 2: explicit admin-only gate for `admin.system`; `scans.create`, `admin.users`, `admin.org` demoted out of read-tier. Default branch is fail-safe (unknown perm → write-tier required). |
| T-30.1-03 | Information disclosure | tools/list response shape for read-scope callers | mitigate | Task 2: read scope now surfaces only `.view` tools. Tool names like `dashboard_create_user` no longer leak into read-scope manifests. |
