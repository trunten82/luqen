# Plan 33-01 — Agent Audit Log Viewer (APER-04)

## Objective

Admins can navigate to `/admin/audit`, filter by date / user / tool / outcome, and paginate through recorded agent tool invocations. Dashboard admins see all orgs; org admins are scoped to their current org.

<action>
1. Extend `AgentAuditRepository` interface's `listForOrg` signature:
   ```ts
   interface AuditListFilters {
     readonly from?: string;       // ISO date
     readonly to?: string;         // ISO date (exclusive)
     readonly userId?: string;
     readonly toolName?: string;
     readonly outcome?: 'success' | 'error' | 'timeout' | 'denied';
   }
   interface AuditListOptions {
     readonly limit?: number;      // default 50, max 500
     readonly offset?: number;     // default 0
   }
   listForOrg(orgId: string | null, filters?: AuditListFilters, options?: AuditListOptions): Promise<AuditLogEntry[]>;
   ```
   Pass `orgId=null` to skip the org filter (dashboard admin, cross-org).

2. Implement in `SqliteAgentAuditRepository.listForOrg`: build a WHERE clause with bound params for each provided filter; `ORDER BY created_at DESC, id DESC LIMIT @limit OFFSET @offset`.

3. Create `packages/dashboard/src/routes/admin/agent-audit.ts`:
   - `GET /admin/audit` — permission gate `requirePermission('admin.system', 'admin.org')`. Parse query params (zod schema). If user has `admin.system`, pass `orgId=null` to repo; else pass `user.currentOrgId`.
   - Reject `?orgId=` query param unless the user has `admin.system` (403).
   - Render `admin/agent-audit.hbs` with `{rows, filters, pagination:{limit, offset, total, next, prev}, users, toolNames}` where `users` and `toolNames` are distinct values for filter dropdowns (SELECT DISTINCT).
   - `GET /admin/audit.csv` — same data surface but emits `content-type: text/csv`, CSV body (RFC 4180 quoted).

4. Create `packages/dashboard/src/views/admin/agent-audit.hbs`:
   - Filter bar at top: date range inputs + user select + tool select + outcome select + submit button (plain form, no HTMX).
   - Data table with the 7 columns from the test. Outcome rendered as `.badge--success | .badge--error | .badge--warning` per outcome.
   - Pagination controls at the bottom using existing `.pagination` pattern.
   - Download CSV link carrying the current filters.

5. Add sidebar link under the admin section for `admin.system` OR `admin.org` holders:
   - `<a href="/admin/audit">{{t "admin.audit.navLabel"}}</a>` in sidebar.hbs.

6. i18n keys in all 6 locales under `admin.audit`:
   - `title`, `navLabel`, `colTimestamp`, `colUser`, `colTool`, `colOutcome`, `colLatency`, `colArgs`, `colDetail`, `filterFrom`, `filterTo`, `filterUser`, `filterTool`, `filterOutcome`, `applyFilters`, `reset`, `downloadCsv`, `noRows`.

7. Register the route in `server.ts`:
   ```ts
   import { agentAuditRoutes } from './routes/admin/agent-audit.js';
   // after proposalRoutes registration:
   await agentAuditRoutes(server, storage);
   ```
</action>
