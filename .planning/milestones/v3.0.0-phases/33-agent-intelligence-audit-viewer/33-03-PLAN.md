# Plan 33-03 — Token Budget + Compaction (AGENT-05)

## Objective

AgentService monitors estimated prompt tokens. When a turn's prompt exceeds 85% of the model's limit, older turns are summarised into a single assistant "summary" message; the tail (last 6 turns) is kept verbatim. Behaviour is feature-gated by `workflow.agent_compaction`.

### Task 2 (GREEN) — implement token-budget + compactWindow

<read_first>

- packages/llm/src/capabilities/agent-conversation.ts (summarisation reuses the same capability via streamAgentConversation)
</read_first>

<action>
1. Create `packages/dashboard/src/agent/token-budget.ts`:
   ```ts
   export const DEFAULT_MODEL_MAX_TOKENS = 8192;
   export const COMPACTION_THRESHOLD = 0.85;
   export const MIN_KEEP_TURNS = 6;

   export function estimateTokens(messages: readonly AgentChatMessage[]): number {
     let chars = 0;
     for (const m of messages) {
       chars += (m.content?.length ?? 0);
       if (m.toolCalls) chars += JSON.stringify(m.toolCalls).length;
     }
     return Math.ceil(chars / 4);
   }

   export function shouldCompact(tokens: number, max = DEFAULT_MODEL_MAX_TOKENS): boolean {
     return tokens > Math.floor(max * COMPACTION_THRESHOLD);
   }
   ```

2. Extend `AgentServiceOptions` with `config.agent_compaction?: boolean` (default true) and `config.modelMaxTokens?: number` (default 8192).

3. In `AgentService.runTurn` BEFORE calling `streamAgentConversation`:
   - Compute `estimateTokens([system, ...windowMessages])`.
   - If `shouldCompact && config.agent_compaction !== false`:
     - Take turns beyond the tail (tail = last MIN_KEEP_TURNS user-initiated turns — reuse the OFFSET 19 idea but with MIN_KEEP_TURNS boundary).
     - Build a summarisation prompt: a synthetic user message asking the model to summarise those older turns into a single terse summary.
     - Call `this.llm.streamAgentConversation({ messages: [...older], tools: [], orgId, userId, agentDisplayName })` with `contextHintsBlock=""` — collect token frames into `summaryText`.
     - Persist a new `role='assistant', status='sent', content: '[summary] ' + summaryText` row via `appendMessage`.
     - Mark older rows `in_window=0` via a new repo method `markOutOfWindow(conversationId, beforeId)`.
     - Re-fetch the window for the real turn.
   - Proceed with the normal turn.

4. Extend `ConversationRepository` interface + SqliteConversationRepository:
   - `getAllMessages(conversationId: string): Promise<Message[]>` — SELECT * FROM agent_messages WHERE conversation_id = ? ORDER BY created_at ASC.
   - `markOutOfWindow(conversationId: string, beforeCreatedAt: string): Promise<void>` — UPDATE agent_messages SET in_window = 0 WHERE conversation_id = ? AND created_at < ? AND status NOT IN ('pending_confirmation', 'streaming').

5. Update `server.ts` AgentService instantiation to pass `config: { agentDisplayNameDefault: 'Luqen Assistant', agent_compaction: true, modelMaxTokens: 8192 }`.
</action>
