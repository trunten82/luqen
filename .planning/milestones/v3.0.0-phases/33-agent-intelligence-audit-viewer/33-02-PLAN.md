# Plan 33-02 — Context-aware Agent (AGENT-04)

## Objective

AgentService injects a `ContextHints` block into the system prompt on every runTurn. The block lists the user's 5 most-recent scans, 10 most-recent active brand guidelines, and 10 applicable regulations so the model can reference them without the user pasting IDs/URLs.

### Task 2 (GREEN) — implement collectContextHints + formatContextHints + prompt integration

<read_first>

</read_first>

<action>
1. Create `packages/dashboard/src/agent/context-hints.ts`:
   ```ts
   export interface ContextHints {
     recentScans: Array<{ id: string; siteUrl: string; status: string; totalIssues: number; detectedAt: string }>;
     activeBrands: Array<{ id: string; name: string; lastScored: string | null }>;
     applicableRegulations: Array<{ id: string; shortName: string }>;
   }
   export async function collectContextHints(storage, {userId, orgId}): Promise<ContextHints>;
   export function formatContextHints(h: ContextHints): string;
   ```
   - `orgId === ''` → cross-org query (no org filter).
   - Wrap each fetch in try/catch → return empty list on failure (defence-in-depth).

2. Modify `packages/llm/src/prompts/agent-system.ts`:
   ```ts
   export function buildAgentSystemPrompt(opts?: { contextHintsBlock?: string }): string {
     return TEMPLATE.replace('{contextHints}', opts?.contextHintsBlock ?? '');
   }
   ```
   Append `\n\n{contextHints}\n` at the end of TEMPLATE (after the "Output capabilities" block added in Plan 32.1-multimodal).

3. Modify `packages/llm/src/capabilities/agent-conversation.ts` `AgentConversationInput`:
   Add optional `contextHintsBlock?: string`. When building `systemContent`, pass through to `buildAgentSystemPrompt({ contextHintsBlock: input.contextHintsBlock })` instead of a bare `buildAgentSystemPrompt()`. Existing callers without the field still work (optional).

4. Modify `packages/dashboard/src/agent/agent-service.ts` `runTurn`:
   - After resolving `agentDisplayName`, call `collectContextHints(this.storage, {userId, orgId})` and `formatContextHints(hints)`.
   - Pass the resulting string as `contextHintsBlock` to `this.llm.streamAgentConversation`.
   - Wrap the collect call in try/catch — log + pass empty string on failure.

5. Extend `AgentStreamInput` (the transport DTO) with optional `contextHintsBlock`. The llm-client serialises it into the POST body; the llm service forwards it into the capability.
</action>
