    ## Tool classification (MANDATORY — no TODOs allowed)

    The compliance service stores jurisdictions, regulations, requirements, proposals, and sources in a SHARED reference database. Per the current DbAdapter schema (confirm at `read_first` of `packages/compliance/src/db/adapter.ts`), none of the compliance tables have an `orgId` column — they are global reference data consumed by every org identically. Every one of the 11 handlers is therefore classified **GLOBAL** (accesses global reference data only, no org filtering needed). Each handler MUST carry a code comment `// orgId: N/A (global reference data)` immediately above the `async (args) => {` line.

    | # | Tool | Classification | Required change |
    |---|------|----------------|-----------------|
    | 1 | compliance_check | GLOBAL | Prepend comment `// orgId: N/A (global reference data)`; read `const _ctx = getCurrentToolContext();` at top of handler (for future phase auditability) but do NOT mutate the call to `checkCompliance()`. |
    | 2 | compliance_list_jurisdictions | GLOBAL | Same pattern: comment + `_ctx` read. `db.listJurisdictions(args)` unchanged. |
    | 3 | compliance_list_regulations | GLOBAL | Same pattern. `db.listRegulations(args)` unchanged. |
    | 4 | compliance_list_requirements | GLOBAL | Same pattern. `db.listRequirements(args)` unchanged. |
