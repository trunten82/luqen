<objective>
Deliver the user-facing chat drawer: floating entry button, right-side panel, message rendering, token-level streaming via plain EventSource, localStorage persistence of open/closed state. The drawer is mounted in the shared authenticated layout so it appears on every page. Matches UI-SPEC Surface 1 end-to-end.

Purpose: AGENT-01 requires users to open a text chat side panel in the dashboard and receive a streamed response without a full page reload. Without this plan, Plan 04's backend routes are unreachable from the browser. This plan produces the minimum viable UX surface; Plan 07 adds confirmation dialog + speech input on top.

Output: agent-drawer.hbs + agent-messages.hbs + agent-message.hbs partials; main.hbs layout injection; ≤200 LOC of scoped CSS; agent.js IIFE with EventSource streaming + localStorage + event delegation; session.ts passthrough for `user.orgAgentDisplayName`; ~28 new i18n keys; Playwright E2E smoke test.
</objective>
