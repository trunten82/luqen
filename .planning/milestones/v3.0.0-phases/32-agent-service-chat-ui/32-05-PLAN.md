| i18n key lookup | Misspelled keys render as literal `{{t}}` placeholders or empty strings — template regression |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-05-01 | Tampering | destructiveTools tool name contains HTML (< , >, script) | mitigate | Handlebars default-escapes interpolated values; tool names are controlled by DASHBOARD_TOOL_METADATA (internal, not user-supplied). Regression risk is zero unless a future dev adds `{{{...}}}` triple-stache. Lint check (Task 4 Test F negative assertion) catches this for the specific override-field absence; admin auditor reviews PRs adding `{{{` |

| T-32-05-03 | Information Disclosure | Locked-fence tooltip leaks implementation detail | accept | Tooltip text (e.g. "enforces per-user role-based access control") is product-documentation-level. Not a secret |
| T-32-05-04 | Spoofing | Per-org-override UI hidden but server accepts orgId | mitigate | Plan 02 Task 2 adds the SERVER-side guard rejecting orgId !== undefined for agent-system. This plan adds the UI-side gate (defense-in-depth). Plan 05 Task 4 Test F asserts the UI absence |
