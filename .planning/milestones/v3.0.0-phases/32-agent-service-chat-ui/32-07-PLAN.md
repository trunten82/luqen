## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE frame → dialog DOM | data.args + data.confirmationText rendered inside dialog |
| Approve/Cancel POST → /agent/confirm|deny | CSRF-protected; server verifies user owns conversation |
| DB pending_confirmation row → panel HTML → client DOM-recovery | Tool call JSON must round-trip without XSS or state loss |
| Web Speech API → textarea | Transcript text — no auto-submit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-07-01 | Elevation of Privilege | Double-approve dispatches tool twice | mitigate | Button disables on first click; server-side state-machine (Plan 04) only transitions pending_confirmation → sent once (409 on second attempt). Test 4 + Task 3 Test B enforce both layers |
| T-32-07-02 | Tampering | XSS via tool arg / confirmationText in dialog | mitigate | Task 2 uses textContent for all dialog population; handlebars escapes {{t "agent.confirm.intro" name=...}} — no raw HTML from server. Regression: `grep innerHTML packages/dashboard/src/static/agent.js` returns only loadPanel server-render swap |
| T-32-07-03 | Spoofing | Confirm POST with wrong messageId (another user's) | mitigate | Server route (Plan 04) verifies message belongs to authenticated user's conversation. Task 3 Test F regression asserts 403 |
| T-32-07-04 | Tampering | CSRF bypass on confirm/deny | mitigate | Fetch includes x-csrf-token header from meta tag; server CSRF middleware validates. Task 3 Test B extends to include a CSRF-absent call returning 403 if time permits |
| T-32-07-05 | Information Disclosure | Web Speech API sends audio to Google servers (EU residency) | accept | Per D-33 — out-of-scope for MVP.
